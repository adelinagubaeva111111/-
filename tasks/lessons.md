## 2026-04-09 — Система обучения Cursor на кейсах

- Лучший рост дает не один "идеальный" промпт, а замкнутый цикл `V1 -> рубрика -> анти-банальность -> V2`: по пилоту 10 кейсов средний балл вырос с 61.5 до 84.1, а красные флаги упали с 2.7 до 0.6 на кейс.

## 2026-04-09 — 30 бесплатных кейсов с разборами (Harvard)

- Для реального качества важен не просто "бесплатный кейс", а пара `кейс + разбор` (practitioner/educator guide или epilogue): это резко снижает риск тренироваться на случайном контенте.
- Под 3-часовой формат лучше всего работают кейсы, где есть конфликт ограничений, несколько стейкхолдеров и необходимость защищать выбор стратегии, а не просто генерировать идеи.
- Нестандартные кейсы (кризис, моральные дилеммы, техно-развилки) полезны, если после них принудительно добавлять бизнес-структуру: метрики, коалиция, сценарии, план 30-60-90.
- Практический запуск: брать из реестра `C011-C020` как батч 1, проходить цикл `V1 -> рубрика -> чеклист -> V2`, фиксировать баллы и красные флаги в индексе.

## 2026-04-09 — Интенсивное обучение "10 кейсов + 1 слепой разбор"

- Реальный прирост дает связка "слепое решение -> пост-мортем по разбору -> калибровка правил": на `C011` прирост с 78 до 87.
- Для бизнес-кейсов блок рынка/ЦА/CJM/конкурентов/unit economics/GTM должен быть обязательным, иначе решение выглядит умно, но не защищается перед жюри.
- Частая скрытая слабость — отсутствие decision rights и legal-compliance: идея может быть сильной, но проваливается на внедрении.
- Следующий батч запускать в том же режиме: `C012-C021`, у каждого кейса фиксировать V1, V2, разницу и 3 ключевые правки.

## 2026-04-09 — Две дополнительные глубокие итерации

- Итерации `C021-C030` и `C031-C040` подтвердили паттерн: креативные инициативы проходят жюри только когда есть доказуемая экономика и архитектура внедрения.
- По самостоятельным кейсам `C023` и `C037` прирост после пост-мортем составил `+8` и `+7`; средний прирост `+7.5`.
- Для нестандартных кейсов критичны дополнительные блоки: `stop-doing list`, `crisis communication`, `impact-per-dollar`, `contract-for-results`.
- Текущая система уже ближе к формату Сколково: не "идеи ради идей", а креатив + реалистичность + бизнес-метрики.

## 2026-04-09 — Презентация pptx в стиле Miro (awesome-design-md)

- В [awesome-design-md](https://github.com/VoltAgent/awesome-design-md) у Miro в папке только README: полный `DESIGN.md` ведёт на [getdesign.md/miro](https://getdesign.md/miro/design-md); для слайдов достаточно публичного описания (жёлтый акцент, тёмный `#050038`, светлый фон).
- На Windows без Python удобно собирать **pptx через COM PowerPoint**; тексты лучше держать в **UTF-8 JSON**, а скрипт `.ps1` — в ASCII, иначе ломается разбор кириллицы. **SaveAs** с кириллицей в имени файла через COM может дать кракозябры — безопаснее латинское имя `.pptx`.
- `$ppt.Visible = 0` на части конфигураций Office даёт ошибку «Hiding the application window is not allowed» — оставляли видимым приложение до `Quit()`.
