# Датасет

Здесь хранится корпус кейсов для тренировки качества решений Cursor.
Один кейс = одна карточка в индексе + при необходимости отдельный файл с условием.

## Правила хранения

- ID формата: `C001`, `C002`, ... `C120`.
- Источник помечать строго: `Harvard`, `OpenMBA`, `Skolkovo`.
- Статус только из списка: `не начат`, `V1`, `V2`, `финал`.
- Для каждого кейса фиксировать минимум: отрасль, тип задачи, уровень сложности, результат оценки.
- Для финального пула обязательно фиксировать 2 ссылки: `кейс` и `разбор`.

## Стандарты разметки

- Тип задачи: `рост`, `эффективность`, `запуск`, `антикризис`, `ценообразование`.
- Сложность: `низкая`, `средняя`, `высокая`.
- Ловушки формулировки: кратко, 1-2 строки.
- Главная метрика: только измеримая (`CAC`, `LTV`, `маржа`, `retention`, `payback` и т.д.).

## Фильтр релевантности для кейс-чемпионата (обязательный)

Кейс включается в основной пул только если:

1. Бесплатный и открытый доступ.
2. Есть разбор официального/академического уровня:
   - practitioner guide,
   - educator guide,
   - epilogue/sequel,
   - академический комментарий из надежного источника.
3. Развивает минимум 2 навыка из списка:
   - декомпозиция,
   - сравнение альтернатив,
   - бизнес-логика и метрики,
   - риск-менеджмент,
   - план внедрения,
   - защита решения от критики.
4. Применим к 3-часовому формату (можно пройти цикл V1 -> V2 за одну сессию).
