# Приложение к инвест-слайду: обоснование оценки по человеко-часам (актуальная версия)

## 1) Что это за расчет

Это расчет бюджета **этапа Product v1 после MVP**, а не бюджета MVP.  
Метод: `bottom-up` (от задач к часам).

Логика:

- берем блоки работ из ТЗ первой очереди;
- делим на подзадачи;
- оцениваем трудоемкость по типу задач;
- отдельно считаем, где ИИ ускоряет работу безопасно;
- фиксируем, сколько часов закрывает команда внутри, а сколько уходит в подряд.

Нормативы:

- простой экран/форма: 6-12 ч
- экран/сценарий с API и состояниями: 16-40 ч
- модуль с бизнес-логикой: 60-180 ч
- интеграция (MOEX/платежи/push): 40-120 ч
- безопасность/ПДн/аудит: 80-160 ч
- QA + стабилизация: 20-30% от dev-часов

## 2) Трассировка: блоки -> смысл -> часы


| Блок работ                              | Что это простыми словами                                                            | Часы |
| --------------------------------------- | ----------------------------------------------------------------------------------- | ---- |
| Архитектура, декомпозиция ТЗ, UX-логика | Проектируем роли, доступы, переходы по экранам, структуру данных и логику сценариев | 220  |
| Backend                                 | API, авторизация, портфель, сделки, подписки, отчеты, серверные правила             | 760  |
| Mobile/Web клиент                       | Пользовательские экраны, состояния, формы, интеграция с API                         | 680  |
| Админка управления контентом            | Техническая панель управления уроками/заданиями (не создание уроков)                | 220  |
| QA и стабилизация                       | Тестирование, регресс, проверка фиксов, доведение до стабильного релиза             | 320  |
| DevOps/инфраструктура                   | CI/CD, окружения, мониторинг, логи, алерты, бэкапы                                  | 180  |
| Наполнение контента                     | Подготовка уроков/карточек/тестов и фактчек                                         | 190  |
| Аналитика/психология                    | Анти-азартные ограничения и проверка мотивационных механик                          | 120  |


**Итого трудоемкость этапа:** **2 690 ч**

## 3) Где ИИ реально экономит часы

- ИИ используем в безопасных зонах: ресерч, черновики контента, микрокопирайт, шаблонный код, тестовые заготовки, документация.
- Критичные зоны не делегируем ИИ: безопасность, ПДн, платежи, юридические формулировки, финальные продуктовые решения.

Эффект:

- часы, где ИИ применим: **1 450 ч**
- средняя экономия: **16%**
- экономия: **230 ч**

## 4) Гибридная модель выполнения

- общая трудоемкость: **2 690 ч**
- внутренний вклад команды: **850 ч**
- экономия за счет ИИ: **230 ч**
- **часы к оплате подрядчикам: 1 610 ч**

## 5) Финансовый расчет (синхронизирован со слайдом)

- средневзвешенная ставка подрядчиков: **1 900 ₽/ч**
- разработка: **1 610 x 1 900 = 3 059 000 ₽**
- инфраструктура и сервисы (7 месяцев): **210 000 ₽**
- юридический/комплаенс минимум: **140 000 ₽**
- AI-инструменты: **90 000 ₽**
- резерв на риски и пилотные доработки: **300 000 ₽**

**Итоговый инвестиционный запрос:** **3 799 000 ₽**  
Для презентации: **3,8 млн ₽**

## 6) Как объяснять это жюри в двух фразах

- "Сумма рассчитана от объема работ в человеко-часах, а не от абстрактных зарплат."
- "Экономия получена за счет управляемого использования ИИ в рутинных задачах, без компромисса по безопасности, качеству и комплаенсу."
