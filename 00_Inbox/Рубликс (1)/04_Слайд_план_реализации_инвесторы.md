### Оценка трудозатрат (человеко-месяцы, «остаток от ТЗ», эксперты исключены)

Ориентир для **полного** доведения до соответствия ТЗ с трёх клиентов + админка (без чата с экспертом):

| Блок | Чел.-мес. (мин–макс) | Пояснение |
|------|----------------------|-----------|
| Backend core (auth, роли, ПДн, связь родитель–ребёнок, аудит) | 2.5–4 | Включая требования 152-ФЗ на базовом уровне |
| MOEX-модуль (нормализация, кэш, деградация при сбоях) | 1.5–3 | Сильно зависит от выбранного API и лимитов |
| Симулятор (инструменты, ордера, портфель, PnL) | 2.5–4.5 | Много краевых случаев |
| Обучение и прогресс (каталог, тесты, привязка к практике) | 2–3.5 | Контент наполняется отдельно по часам методиста |
| Геймификация (уровни, миссии, челленджи, рейтинги) | 1.5–3 | Пуши и антиабьюз |
| Клиенты mobile (оба стора) | 5–9 | Нижняя граница при сильном переиспользовании прототипа и одном стеке |
| Web-клиент MVP | 1.5–3 | Параллельно с mobile после стабилизации API |
| Админка | 1.5–3 | Контент, тарифы, модерация |
| Платежи / подписки (веб по ТЗ) | 1–2 | Провайдер, вебхуки, восстановление покупок/доступа |
| Аналитика, crash reporting, мониторинг | 0.5–1 | |
| Итого инженерия (суммарно) | **20–36** | Диапазон = неизвестный % готовности прототипа + стек |

**Premium «ИИ-коуч» в ТЗ:** в базовую оценку не заложен полноценный LLM с модерацией; это отдельно **+0.5–2 чел.-мес.** при выборе готового API и жёстких лимитов. На слайде можно: *«персонализация v1 — правила/контент; ИИ-коуч v2 — после метрик»*.

**Сжатие после аудита прототипа:** ориентир **минус 20–35%** к нижней части диапазона, если подтверждено переиспользование ядра симулятора, auth и части экранов.

### Деньги (оценка ФОТ подряда/найма, без НДФЛ/налогов работодателя — уточнять у бухгалтерии)

Базовая ставка для расчёта: **200–280 тыс. руб./мес** на одного инженера «в руки» или на подряд (зависит от договора). Возьмём **240 тыс. руб./мес** как среднюю для итога.

| Сценарий | Чел.-мес. инженерии | ФОТ инженерии (240k × мес.) |
|----------|---------------------|-----------------------------|
| Оптимист (много от прототипа) | ~14–18 | ~3,4–4,3 млн руб. |
| База (середина ТЗ) | ~22–26 | ~5,3–6,2 млн руб. |
| Консерватив (натив, сложный MOEX, мало reuse) | ~30–36 | ~7,2–8,6 млн руб. |

**Неинженерные статьи (добавить к ФОТ):**

| Статья | Ориентир, руб. |
|--------|----------------|
| Дизайн (доп. к инженерным мес.) | 300–700 тыс. |
| Юридическое сопровождение (ПДн, оферты, договоры) | 250–600 тыс. |
| Инфраструктура (облако РФ, пуши, аналитика, домены) 12 мес. | 200–500 тыс. |
| Резерв рисков **15–20%** от подытога | считается от суммы выше |
| Грант Альфа-Банка **500 тыс. руб.** | вычесть из потребности или показать строкой «привлечено» |

**Пример итога «база» (иллюстрация, не договор):**  
Прямые статьи: ФОТ ~5,5 млн + дизайн ~0,5 млн + юр ~0,4 млн + инфра ~0,35 млн. Итого прямых статей�� **~6,75 млн руб.** С резервом 18%: **~8,0 млн руб.** до учёта гранта.  
После гранта 500 тыс.: **~7,5 млн руб.** чистой потребности (порядок величины).

*Итог заменяется на смету после оценки разработчика по задачам.*

## Короткий текст для вставки одним блоком (если слайд один)

**Заголовок:** План реализации и бюджет разработки  

**Тезисы:**

1. Продукт по ТЗ: iOS, Android, веб, единый backend, интеграция с MOEX, обучение, симулятор, геймификация, родительский кабинет, админка, подписки — **без** запуска экспертных консультаций и чата с экспертом на первом этапе.

4. Грант **500 тыс. руб.** учитывается в покрытии расходов; **не дублируем** в запросе отдельную строку «MVP» — доведение до MVP команда закрывает своими силами.
