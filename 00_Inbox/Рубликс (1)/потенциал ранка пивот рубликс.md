# Потенциал ранка пивот рубликс

## 1) Что считаем и зачем

Документ фиксирует подтвержденные входные данные для B2B-пивота `Рубликс -> банк`, показывает формулы и итоговую экономику в двух моделях:

- модель A: фикс/CPA за активного подростка;
- модель B: процентная модель (от целевого CAC и отдельно от кешбэк-пула).

Все расчеты приведены к единой шкале `1 месяц`.

## 3) Принятая механика пивота для расчета

1. Подросток тратит деньги по карте банка-партнера.  
2. На целевые категории начисляется `5%` кешбэк.  
3. Кешбэк удваивается в виртуальной валюте Рубликс.  
4. Реальная выплата удвоенного кешбэка идет только при выполнении условий (уроки/прогресс).

---

## 4) Формулы (1 активный подросток, 1 месяц)

Обозначения:

- `S` — среднемесячные траты подростка, ₽
- `a` — доля трат в целевых категориях (еда/транспорт/книги/канцтовары)
- `r` — ставка кешбэка (в модели `5%`)
- `q` — доля пользователей, выполнивших условия для реальной выплаты
- `C_media` — медиа+промо стоимость активного пользователя
- `C_ops` — операционные затраты на активного (аналитика/антифрод/поддержка)
- `P_bank` — цена, которую банк платит за активного

Формулы:

- Базовый кешбэк-пул: `B = S x a x r`
- Удвоенный пул: `D = 2 x B`
- Фактическая выплата: `C_cash = D x q`
- Полная себестоимость: `C_total = C_media + C_cash + C_ops`
- Доход Рубликс (фикс-модель): `Revenue_fix = P_bank - C_total`
- Доход Рубликс (процент от CAC): `Revenue_%CAC = CAC_target x k`
- Доход Рубликс (процент от пула): `Revenue_%pool = Pool x p`

## 5) Проверка C_media (почему 1050 ₽ реалистично/нереалистично)

### 5.1 Формула

`C_media = CPI / CR_install_to_active`

Где:

- `CPI` — стоимость установки;
- `CR_install_to_active` — доля установок, ставших активными пользователями.

### 5.2 Откуда берем CPI и CR

- Локальные ориентиры CPI в РФ по кейсам VK Ads: `~88-117 ₽` за установку  
  Источники:  
  [Кейс 117 ₽ CPI](https://vc.ru/marketing/1727179-4000-ustanovok-prilozheniya-po-117-rublei-iz-vk-reklamy-keis-internet-magazina)  
  [Кейс 88.57 ₽ CPI](https://klimchukovy.ru/case_vk_ads)
- Глобальный benchmark для finance app: median eCPI `1.21$`  
  Источник: [Adjust/AppLovin release, 2024](https://investors.applovin.com/news/news-details/2024/Adjust-Finds-119-YoY-Revenue-Increase-In-Finance-Apps-Globally-Indicating-Sustained-Sector-Growth-Through-2024-2024-hI7qsW7w7H/default.aspx)
- Финтех retention-benchmark как прокси для активных: D30 `7.8%-14.2%`  
  Источник: [AppsFlyer summary](https://www.rocketshiphq.com/appsflyer-state-of-app-marketing-2025-summary/)

### 5.3 Сценарный расчет C_media

- Оптимистично: `90 / 14.2% = 634 ₽`
- База: `105 / 10% = 1050 ₽`
- Консервативно: `120 / 7.8% = 1538 ₽`

Вывод:

- `1050 ₽` — это реалистичная **середина диапазона** при CPI около 100-110 ₽ и конверсии install->active около 10%.
- Нереалистично низким `1050` становится, если фактический CR ниже 10% или CPI уходит выше 120 ₽.

## 6) Базовый расчет экономики (месячный)

Вход:

- `S = 5280 ₽` (SuperJob)
- `a = 60%` (базовый сценарий целевых MCC)
- `r = 5%`
- `q = 50%` (база; сценарий ниже)
- `C_media = 1050 ₽` (из раннего расчета CPI->active)
- `C_ops = 120 ₽` (минимальный операционный слой пилота)

### 6.1 Кешбэк-часть

- `B = 5280 x 0.6 x 0.05 = 158.4 ₽`
- `D = 316.8 ₽`
- `C_cash = 316.8 x 0.5 = 158.4 ₽`

### 6.2 Полная себестоимость

- `C_total = 1050 + 158.4 + 120 = 1328.4 ₽`

Итого база: `~1328 ₽` на 1 активного в месяц.

### 6.3 Сценарии полной себестоимости

- Оптимистичный: `~809 ₽`
- Базовый: `~1328 ₽`
- Консервативный: `~1940 ₽`

(сценарии учитывают диапазоны медиа, q и ops).

## 7) Модель A — фикс/CPA от банка

Формула:

`Revenue_fix = P_bank - C_total`

При `C_total ~1328 ₽`:

- если `P_bank = 1400 ₽` -> `+72 ₽`
- если `P_bank = 1600 ₽` -> `+272 ₽`
- если `P_bank = 1800 ₽` -> `+472 ₽`
- если `P_bank = 2000 ₽` -> `+672 ₽`

Вывод по фикс-модели:

- точка безубыточности в базе: `~1330 ₽`;
- комфортный рабочий уровень: `1600-2000 ₽` за активного;
- ниже `1400 ₽` модель очень уязвима к росту промо/медиа.

---

## 8) Модель B — процентная

### 7.1 Процент от целевого CAC (рабочий вариант)

Формула:

`Revenue_%CAC = CAC_target x k`

Где `k` — ваш процент.

Реалистичный диапазон для переговоров: `15-30%`, рабочая зона: `18-28%`.

Примеры:

- `CAC=1400` -> 20% = `280 ₽`, 25% = `350 ₽`
- `CAC=1600` -> 20% = `320 ₽`, 25% = `400 ₽`
- `CAC=1800` -> 20% = `360 ₽`, 25% = `450 ₽`

Комментарий: в базе при полной себестоимости ~1328 ₽ модель чистого процента работает только если процентная выплата комбинируется с фиксом или если значимую часть промо берет на себя банк.

### 7.2 Процент от кешбэк-пула (как отдельный запрос)

Если процент брать только от пула:

- от `B=158 ₽`: даже 30% = `47 ₽`
- от `D=317 ₽`: даже 30% = `95 ₽`

Вывод: процент только от кешбэк-пула не может быть основным источником выручки Рубликс при текущем уровне трат подростка.

## 9) TAM / SAM / SOM по пивоту

### 9.1 TAM (весь рынок)

Численность 12-17 лет в РФ: `~9.2 млн` человек.  
Источник: [Statista с разбивкой по возрастам и полу (по РФ)](https://www.statista.com/statistics/1100728/number-of-children-by-age-and-gender/)

### 9.2 SAM (доступный сегмент для текущей модели)

Для текущих расчетов трат есть подтвержденный чек именно для 15-17 лет (SuperJob), поэтому операционный SAM считаем как возраст 15-17:

- 15 лет: `1.723 млн`
- 16 лет: `1.622 млн`
- 17 лет: `1.491 млн`
- Итого `SAM_15_17 = 4.836 млн`

Источник возрастных численностей: [Statista](https://www.statista.com/statistics/1100728/number-of-children-by-age-and-gender/)  
Источник трат 15-17: [SuperJob](https://www.superjob.ru/research/articles/115273/karmannye-dengi-dlya-shkolnikov/)

### 9.3 Денежная база TAM и SAM

- Траты на 1 подростка (15-17): `5280 ₽/мес`
- TAM (грубая верхняя оценка, если применять этот чек ко всем 12-17):

`TAM_spend_year = 9.2 млн x 5280 x 12 = 582.9 млрд ₽`

- SAM (более строгая оценка, 15-17):

`SAM_spend_year = 4.836 млн x 5280 x 12 = 306.4 млрд ₽`

Если в целевые MCC попадает 60%:

`SAM_target_spend_year = 306.4 x 0.6 = 183.8 млрд ₽`

Базовый кешбэк-пул 5%:

`SAM_cashback_pool_year = 183.8 x 0.05 = 9.2 млрд ₽`

Удвоенный виртуальный пул:

`SAM_double_pool_year = 18.4 млрд ₽`

### 9.4 SOM-сценарии по активной базе Рубликс (от SAM 15-17)

Берем долю проникновения в SAM (15-17):

- 0.5% от 4.836 млн = `24 180` активных
- 1.0% = `48 360` активных
- 2.0% = `96 720` активных

Пример выручки Рубликс в фикс-модели (база `+272 ₽` маржи на пользователя, то есть P_bank=1600):

- 24 180 x 272 = `6.6 млн ₽/мес`
- 48 360 x 272 = `13.2 млн ₽/мес`
- 96 720 x 272 = `26.3 млн ₽/мес`

## 10) Итог для принятия решения

1. Экономика должна считаться только с полной себестоимостью: `медиа + кешбэк + ops`.  
2. Модель «только процент от кешбэк-пула» слабая по деньгам.  
3. Рабочие варианты:
   - фикс/CPA;
   - гибрид: фикс + процент от целевого CAC;
   - процент от кешбэк-пула можно оставить как дополнительный бонусный слой.
4. Для пилота с банком лучше защищать договор как:
   - фикс за активного + KPI на качество (retention/обучение),
   - и отдельно прозрачный калькулятор себестоимости и эффективности.

## 11) Короткая выжимка по цифрам

- `C_media = 1050 ₽` реалистично как база, потому что это середина сценария `634-1538 ₽` по формуле `CPI / CR_active`.
- Полная базовая себестоимость на 1 активного в месяц: `~1328 ₽` (`медиа 1050 + кешбэк 158 + ops 120`).
- В фикс-модели точка безубыточности: `~1330 ₽`; комфортная зона: `1600+ ₽`.
- Модель процента от кешбэк-пула как основная монетизация не работает (слишком маленькая база).
- Рабочий вариант процента: от целевого CAC, обычно в диапазоне `18-28%` (или гибрид с фиксом).
- `TAM 12-17`: `~9.2 млн`; `SAM 15-17`: `~4.836 млн`.
- `SAM_spend_year`: `~306.4 млрд ₽`; `SAM_cashback_pool_year`: `~9.2 млрд ₽`.
- SOM (от SAM): 1% активной базы = `48 360` пользователей.
